# TaskTimer

Десктопный таймер задач на Python + [PySide6](https://doc.qt.io/qtforpython/) (Qt). Помогает вести задачи по дням, засекать затраченное на каждую время и не выпадать из фокуса. Основная цель — Windows (собирается в один `TaskTimer.exe`), но приложение кроссплатформенное и запускается также на macOS и Linux.

## Возможности

- **Задачи по дням.** Создавайте задачи, запускайте таймер «Старт», ставьте на паузу «Стоп». Справа видна текущая активная задача и накопленное за сегодня время.
- **Завершение и возобновление.** Завершённые задачи можно открыть заново или спрятать фильтром «Только незавершённые».
- **Напоминание «продолжать?»** Через заданный интервал (по умолчанию 40 минут) приложение спрашивает, работаете ли вы ещё над задачей. Если не ответить — таймер остановится. Интервал меняется в *Настройки → Параметры…*
- **История времени.** У каждой задачи есть история интервалов: просмотр, ручная правка, добавление записи за прошедшее время, удаление.
- **Режим концентрации.** Отдельная вкладка «Фокус» с обратным отсчётом (5–40 мин) — напоминание сосредоточиться, не привязанное к задаче.
- **Трей и плавающий виджет.** Окно сворачивается в системный трей. В свёрнутом состоянии поверх всех окон появляется маленький полупрозрачный виджет с текущей задачей и временем, с кнопками старт/стоп; его можно перетаскивать, двойной щелчок разворачивает окно.
- **Интеграция с Битрикс24 (новое).** По вебхуку: импорт проектов и задач с портала, создание задачи прямо в Битрикс24, автосинхронизация завершения/возобновления и **передача затраченного времени** — выбранные интервалы истории сливаются в одну запись на портале. Подробности — в разделе [Интеграция с Битрикс24](#интеграция-с-битрикс24) ниже.

## Интеграция с Битрикс24

Начиная с этой версии приложение умеет синхронизироваться с Битрикс24 по **входящему вебхуку**. Интеграция полностью опциональна: без настроенного вебхука всё работает локально.

### Что нужно сделать на портале

1. В Битрикс24: **Приложения → Разработчикам → Другое → Входящий вебхук**.
2. Выдайте вебхуку права доступа:
   - **Задачи** (`task`) — импорт задач, создание задач, передача времени, завершение/возобновление;
   - **CRM** (`crm`) — проекты (СПА) и поиск компаний, журнал работ по проекту;
   - **Пользователи** (`user`) — определение текущего пользователя (автора записей).
3. Скопируйте URL вида `https://ваш-портал.bitrix24.ru/rest/1/<токен>/`.
4. В приложении: **шестерёнка (Настройки) → вставьте URL → «Проверить соединение»**. При успехе строка станет зелёной и покажет ваше имя. Нажмите **ОК**.

> URL вебхука — это секрет (как пароль): он даёт доступ к порталу. Не передавайте его посторонним. Хранится локально в настройках приложения.

После настройки становятся доступны: **«Выбрать с портала»** (импорт проектов/задач), чекбокс **«Создать задачу в Битрикс24»** в окне новой задачи, кнопка **«Открыть в Б24»** в строке связанной задачи и **«Передать в Битрикс»** в окне истории.

### Логика сохранения результата: слияние истории в одну запись

Учёт времени в приложении ведётся по **интервалам** («сессиям») — каждый запуск/остановка таймера или ручная правка создаёт отдельную запись в истории задачи. При передаче в Битрикс24 эти интервалы **не отправляются по одному**, а **сливаются в единую запись**. Алгоритм (окно **«История» → отметить галочками → «Передать в Битрикс»**):

1. **Выбор интервалов.** Берутся только интервалы, **отмеченные галочками** и **ещё не переданные** ранее (у переданных в колонке «Передано» стоит id записи — они в новую передачу не попадают, поэтому повторная отправка не создаёт дублей).
2. **Слияние.** Длительности выбранных интервалов **суммируются** в одно итоговое время:
   `итог = Σ длительностей выбранных непереданных интервалов`.
3. **Название.** Приложение запрашивает название записи (по умолчанию — название задачи); оно идёт в комментарий записи на портале.
4. **Отправка одной записью** — в зависимости от того, с чем связана задача:
   - **связана с задачей Битрикс24** → `task.elapseditem.add`: учёт времени задачи, итог в **секундах**;
   - **связана с проектом (СПА, entityTypeId 1092)** → `crm.item.add`: запись в журнале работ, итог в **часах** (округление до сотых), дата записи — **последняя дата** среди выбранных интервалов, автор — текущий пользователь вебхука.
5. **Отметка о передаче.** Все слитые интервалы помечаются полученным **id записи** (колонка «Передано»). Так гарантируется, что одно и то же время не уйдёт на портал дважды.

Итог: на портале появляется **одна аккуратная запись** с суммарным временем вместо россыпи мелких интервалов, а локальная история остаётся детальной (по интервалам) и помечает, что именно уже выгружено.

> Завершение и возобновление задач, связанных **с задачей** на портале, синхронизируются автоматически (`tasks.task.complete` / `tasks.task.renew`). Проекты так не закрываются — у них свои стадии в Битрикс24.

Пошаговую инструкцию для пользователя см. в [`ИНСТРУКЦИЯ.md`](ИНСТРУКЦИЯ.md).

## Требования

- Python **3.10+** (PySide6 6.11 не поддерживает Python 3.9)
- PySide6 6.11.0 (см. [`requirements.txt`](requirements.txt))

## Запуск из исходников

**Windows (PowerShell):**

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python app.py
```

**macOS / Linux:**

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python app.py
```

Точка входа — [`app.py`](app.py), которая вызывает `win_timer_app.main:main`.

## Сборка `TaskTimer.exe` (Windows)

В репозитории есть готовый PyInstaller-спек [`TaskTimer.spec`](TaskTimer.spec) и скрипт сборки:

```powershell
.\build_exe.ps1
```

Скрипт ставит зависимости (`requirements.txt` + [`requirements-build.txt`](requirements-build.txt)) и запускает PyInstaller. Результат — один файл `dist\TaskTimer.exe` без консоли; Python на целевой машине не нужен. Подробности для конечных пользователей — в [`README-DISTRIBUTION.txt`](README-DISTRIBUTION.txt).

## Тесты

Логика покрыта тестами на pytest (модели, хранилище, контроллер):

```bash
pip install -r requirements-dev.txt
pytest
```

## Структура проекта

```
app.py                      # точка входа
win_timer_app/
  main.py                   # инициализация QApplication и главного окна
  controller.py             # бизнес-логика: задачи, сессии, таймеры, напоминания (AppController)
  main_window.py            # весь UI: главное окно, диалоги, трей, плавающий виджет
  models.py                 # модели данных: Task, Session, TaskStatus
  storage.py                # загрузка/сохранение состояния в JSON
tests/                      # pytest: test_models, test_storage, test_controller
TaskTimer.spec              # конфигурация сборки PyInstaller
build_exe.ps1               # скрипт сборки exe
```

## Хранение данных

Задачи и настройки сохраняются автоматически в `data.json` в пользовательском каталоге приложения (через `QStandardPaths.AppDataLocation`; на Windows — в профиле пользователя). Если каталог недоступен для записи, используется запасной вариант — `./.localdata/data.json` рядом с приложением.
