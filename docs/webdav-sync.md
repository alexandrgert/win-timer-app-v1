# WebDAV-синхронизация TaskTimer link B24

> **Для пользователей:** раздел [«Синхронизация через облако»](../ИНСТРУКЦИЯ.md) в `ИНСТРУКЦИЯ.md`.  
> Ниже — техническое описание для разработчиков.

Как приложение синхронизирует `data.json` с облаком по WebDAV (Nextcloud, Яндекс.Диск и др.).

## Файлы на сервере

| Файл | Назначение |
|------|------------|
| `tasktimer/data.json` | База задач и UI (без секретов) |
| `tasktimer/data.sync-meta.json` | Метаданные версии: SHA-256, revision, device_id |

Локально (не синхронизируются): `~/.config/tasktimer/webdav.json`, `bitrix.json`.

## Алгоритм pull (скачать и объединить)

1. Скачать `data.json` и `.sync-meta.json` (если есть).
2. **Conflict detection:** если `content_hash` на сервере ≠ `last_remote_content_hash` в локальном `webdav.json`, зафиксировать конфликт.
3. **Merge** локального и удалённого файла:
   - задачи по `id`;
   - при совпадении `id` побеждает задача с **большим числом sessions**; при равенстве — более поздний `created_at`;
   - UI берётся из «самого полного» файла (больше задач / размер).
4. **Normalize:** не более **одной** `running`-задачи; остальные переводятся в `paused` (побеждает задача с более поздним `started_at` активной сессии).
5. Сохранить локально, обновить `last_remote_content_hash`.

## Алгоритм push (загрузить)

**Pull-before-push** — перед upload всегда выполняется merge с сервером (если файл существует), затем upload.

1. Если на сервере есть `data.json` — скачать, проверить hash (conflict detection), merge с локальным.
2. Normalize running (как при pull).
3. `PUT data.json` + `PUT data.sync-meta.json` с новым SHA-256 и revision (meta — до 3 попыток; при сбое — повтор полного цикла data+meta).
4. Обновить `last_remote_content_hash` в `webdav.json`.

Так снижается риск затирания изменений с другого устройства без предварительного merge.

## Ограничения merge

- Сессии **одной и той же задачи** (`id`) **не объединяются** — побеждает целиком одна копия задачи. Если на двух ПК одна задача набирала **разные** интервалы под одним `id`, часть интервалов может быть потеряна. Рекомендуется не запускать таймер одной задачи одновременно на двух устройствах.
- Webhook и пароль WebDAV **никогда** не попадают в `data.json`.

## Сбои и оповещения

- Проверка наличия файла на сервере: `HEAD`; при **405/501** — fallback `GET` с `Range: bytes=0-0`.
- Ошибка при **старте** (если включён sync on startup): запись в `webdav.json` → `last_error`, **уведомление в трее**.
- Ошибка при **выходе**: запись в лог (`logging`) и `last_error`; save/backup всё равно выполняются.
- Конфликт hash: merge + уведомление «объединены конфликтующие изменения».
- **Частичный upload:** при сбое meta после data выполняется повторный цикл (data + meta). Если sync всё равно не удался, `last_remote_content_hash` **не** обновляется (`last_error` в `webdav.json`). При pull, если hash в meta ≠ hash файла, используется hash **файла** (meta считается устаревшим).

## Безопасность

### Права на локальные секреты (`chmod`)

Файлы `webdav.json` и `bitrix.json` записываются через `secure_files.write_json_secrets`:

- файл: **0600** (только владелец);
- каталог `~/.config/tasktimer/`: **0700** (best effort).

Если `chmod` недоступен (некоторые FS / Windows mount), ошибка **логируется**; рекомендуется проверить права вручную: `chmod 700 ~/.config/tasktimer && chmod 600 ~/.config/tasktimer/*.json`.

### TLS / MITM

WebDAV использует системную проверку SSL через `urllib` (стандартные CA). Для self-hosted серверов с собственным CA сертификат должен быть добавлен в системное хранилище. **Certificate pinning не реализован.**

### Терминал и сигналы

По умолчанию приложение перехватывает `SIGINT`/`SIGTERM` для корректного save + WebDAV push.

| Режим | Поведение |
|-------|-----------|
| GUI (по умолчанию) | SIGINT → save, backup, WebDAV push, quit |
| `TASKTIMER_TERMINAL_MODE=1` | обработчики сигналов **не** устанавливаются; `Ctrl+C` ведёт себя как у обычного процесса Python (traceback / немедленный выход) |

Пример отладки из терминала:

```bash
TASKTIMER_TERMINAL_MODE=1 ./run.sh
```

## Ручная синхронизация в UI

Кнопки «Скачать и объединить» / «Загрузить сейчас» работают **без** включения чекбокса «Включить синхронизацию WebDAV» (достаточно URL и логина).

Кнопка **⇅ WebDAV** на главном экране (десктоп) / иконка облака (Android) выполняет полный цикл: pull + merge + push.

## Периодическая проверка (`sync_interval_minutes`)

Если интервал > 0 и WebDAV включён, каждые N минут выполняется **только проверка** удалённого hash (через `.sync-meta.json`, при отсутствии meta — скачивание `data.json`). На **десктопе** проверка работает и когда окно свёрнуто в трей; на **Android** — в фоне (уведомление, если приложение не на экране; WorkManager — не реже 15 мин, если процесс выгружен).

При обнаружении изменений с другого устройства (`content_hash` ≠ `last_remote_content_hash`) показывается запрос **«Скачать и объединить?»**. Автоматического merge нет.

### «Позже» и повторное напоминание

Настройка **`sync_remind_later_minutes`**: 5, 10, 15, 30 или 60 минут. После «Позже» для **той же** версии на сервере запрос повторится через выбранный интервал (`pending_remote_hash` + `pending_remote_remind_at` в `webdav.json`). Если на сервере появилась **новая** версия (другой hash), таймер «Позже» сбрасывается и диалог показывается сразу.

После pull UI вызывает `apply_loaded_state()`: rollover плана, normalize running, пересборка `next_reminder_at` для напоминаний.
